Specifications

Design System
Based on Discord Bot Admin design system with custom CSS properties. Supports dark theme with carefully selected color palette for readability and visual hierarchy.
Technology
Self-contained HTML files with embedded CSS and vanilla JavaScript. No external dependencies or frameworks. Full support for modern browsers (Chrome 90+, Firefox 88+, Safari 14+).
Accessibility
WCAG 2.1 AA compliant. Includes semantic HTML, proper form labels, keyboard navigation, focus states, ARIA attributes, and min 44px touch targets on mobile devices.
Responsiveness
Mobile-first responsive design. Desktop prototypes adapt to smaller screens. Mobile prototype optimized for touch interaction with proper viewport scaling and safe area support.
Interactions
Mode switching, preset application, real-time slider updates, character counting, text selection toolbar, modal dialogs, collapsible sections, toast notifications, and smooth animations (200ms standard transitions).
Colors
Primary Accent: #cb4e1b (Orange) | Secondary Accent: #098ecf (Blue) | Background: #1d2022 | Text: #d7d3d0

Key Features

Mode System
Simple: Basic message textarea and voice selection | Standard: + Presets, speaking styles, intensity control | Pro: + SSML preview panel with copy functionality
Quick Presets
8 quick presets: Excited, Announcer, Robot, Friendly, Angry, Narrator, Whisper, and Shouting. Each uses Heroicons. Instantly apply preset styles with toast confirmation.
Voice Management
Multiple voice options (Jenny, Nova, Guy, Sonia, Ryan). Speaking styles dynamically filter based on voice compatibility. Intensity slider (Subtle 0.5 to Intense 2.0) with labeled positions.
Text Enhancement
Floating toolbar for text formatting. Apply strong emphasis (orange), moderate emphasis (blue), pause markers (gray), or clear all formatting. Preview shows visual indicators.
SSML Support (Pro)
Syntax-highlighted SSML preview panel showing generated markup. Collapsible section. Copy-to-clipboard functionality. Example includes prosody, emotion, and emphasis tags.

Usage Notes

Testing Desktop
Open tts-portal-desktop.html in browser at 1200px+ width. Use browser DevTools to test responsive breakpoints. Toggle between modes (Simple, Standard, Pro) to see feature changes.
Testing Mobile
Open tts-portal-mobile.html on mobile device or use DevTools device emulation (iPhone/Pixel). Test touch interactions, collapsible sections, and horizontal preset scrolling.
Emphasis Toolbar
Open emphasis-toolbar.html. Select text in the textarea to see floating toolbar above selection. Click formatting buttons to apply styles. Preview updates in real-time below.
Component Reuse
Extract components from prototypes for production use. Move CSS to shared stylesheets. Extract JS to modular functions. Integrate with backend API for real TTS generation and preview.
