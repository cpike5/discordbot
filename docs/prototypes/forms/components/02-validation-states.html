Validation States

Input Validation States

Validation states provide visual feedback to users about the status of their input. These states include default, hover, focus, error, success, warning, and disabled. All states follow accessibility guidelines by not relying on color alone.

Error State

Error states indicate invalid input. Always include an icon and descriptive text message, never rely on color alone. Use aria-invalid="true" and role="alert" for accessibility.

Please enter a valid email address (e.g., user@example.com).
Password must be at least 8 characters and include uppercase, lowercase, and a number.
  • Must be at least 3 characters long
  • Cannot contain special characters

Success State

Success states confirm valid input. Use to provide positive reinforcement, especially for fields requiring specific formats or availability checks.

This username is available!
Email format is valid. A confirmation will be sent.

Warning State

Warning states indicate potential issues that don't prevent submission but require attention. Use for suggestions, deprecation notices, or approaching limits.

Password is weak. Consider using a stronger password.
Approaching character limit. 145/160

Info State

Info states provide neutral information or context about a field without indicating errors or success.

You can find your token in the Discord Developer Portal.
